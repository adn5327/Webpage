Motivation

Welcome to mini valgrind wonderland!

This week, you will have some fun implementing a mini version of valgrind. As you know, valgrind monitors memory usage (calls to malloc and free) and detects memory mismanagement. For this lab, you will need to implement a memory leak detector “mini_valgrind” which prints out a summary of the memory leaks in the chosen C program. Memory leak occurs when a computer program allocates memory, but doesn't release the memory when it exits. As a student of CS241, you have probably encountered memory leak problems before and now you have the chance to implement a program that detects memory leak! Isn’t that cool?

We have provided you mini_valgrind.h file which contains: 

struct _meta_data
which contains the address of the pointer malloc returns, the size of blocks allocated, line number in the C program where the leaked memory was allocated, the name of the C program, and a pointer to the next node.

Your job is to implement the functions in mini_valgrind.c. After running the program, you will receive a summary of memory leak information in the C program in result.txt. You can add more test cases in test.c.

Demo

Your section leaders will demo a working implementation of this text editor.

Overview

You will be making your own simple text editor in C!
Your editor can be run like any other editor as: 
./editor <filename>
Run your editor program with the file name provided as input and store the contents of the file in a data structure. Your program should accept commands from the user in an interactive manner (from stdin). Make sure you can perform the following basic tasks from your editor program:
  1. Write text to a file at a specified line number.
  2. Find text in a file and display the matching text with the line number.
  3. Display the contents of a file.
  4. Save the file.
Nota Bene: Lines will be 1 indexed for this assignment! Send all your debug printing to stderr instead of stdout.

Reading From Standard Input

Your text editor needs to be able to parse arguments and read from standard input.
You need to parse arguments because a user will launch your editor with the following command: 
./editor [filename]
So you will need to be able to get the name of file from argv[] and open it. If the file does not exist, then just make it when the user saves (have an empty data structure for now).
You will also need to be able to read from standard input for when the user gives your text editor a command like:
w 3 I like cats

Data Structure

Your text editor needs a data structure to store the contents of the file.
How you implement this is completely up to you and you should decide upon this based on your knowledge of C strings. Note that we have provided an empty 'init' and 'destroy' function that you should use to initially load your data structure when your text editor is launched and destroy when a user inputs the quit command. Note that your init should malloc the fields inside your data structure and destroy should free all memory.
Please keep the following in mind when making your data structure
  1. Lines are 1 indexed
  2. There is no limit on the number of lines
  3. There is no limit on the number of characters perline
  4. Some lines may be empty

Printing Content

Your text editor should be able to print out the content of its data structure.
After a user has launched your editor, they can ask it to print out the contents of its data structure by entering commands into standard input. There are two ways your user can ask your editor to do this. The first is by asking you to print the entirety of your data structure to standard out with the following command: 
p
Which will print the file (that here has 3 lines) like so: 
1    This is a sentence.
2    Is this a question?
3    I like cats!
And the second way is by asking you to print out just one line with the following command: 
p 3
which should print out just line 3 (if line 3 does not exist then this is a no op; if line 3 is empty then still print the line number and tab) like so 
3    I like cats!
There is a tab inbetween the line number and the line ('\t' is the character escape for tab).

Write Text To A File

There are two ways that your editor should be able to write to a file once it is launched.
The first way to write to a file is to OVERWRITE a line completely with the following command: 
w 3 I like cats!
This will OVERWRITE line 3 with "I like cats!". The second way to write to a file is to APPEND a line with the following command: 
a 3 I like cats!
This will APPEND to line 3 "I like cats!". An extra feature your editor needs to account for is a newline escape (a backslash followed by lowercase n). For example the user can input the following: 
w 3 I like cats!\nDogs are alright.
This will OVERWRITE line 3 with "I like cats!". Second, because there is a newline escape (the backslash n), this will ALSO create a NEW line that will become line 4 and read "Dogs are alright.". The old line 4 and all subsequent lines after get shifted down. Both the "w" and "a" modes may contain multiple newline escapes, and an additional line should be inserted for each.

Please note that all these write operations will only modify your datastructure and not the actual file. Only when the user runs the save command will you write the changes over to the actual file. Also note that if your editor is asked to write to a line that does not exist yet, then it should fill in the gap with empty lines. For example if you are asked to write to line 10 in a 3 line file, then lines 4 to 9 should be empty lines.

Finding Text

Your text editor should be able to find all occurrences of a word and display the words with their line numbers.
Remember the searching should be done on the data structure and not the actual file. So if your user has made some changes since they opened the file, then your editor should be able to find the words in those changes. If your user inputs the following into standard input: 
/like cats
Then your texteditor should print all occurances (the whole line) with their line numbers in following format. 
3    I [like cats]!
There is a tab inbetween the line number and the line. You will need to surround the search term with square brackets.

Saving Text

Your text editor should be able to save back all the changes to the file.
So if your user inputs the following into standard input: 
s
Then your text editor should write back everything to the file.

Closing Your Text Editor

Your text editor should be able to close without memory leaks.
So if your user inputs the following into standard input: 
q
Then your text editor should quit and free your data structure (even if there are changes the user has not saved).
